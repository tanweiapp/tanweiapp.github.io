React-虚拟DOM

  • 虚拟DOM 其实就是一个用来描述真实DOM的js对象；
渲染步骤：
  • 1 生成state 数据
  • 2 生成jsx模版
  • 3 数据 + 模版 生成虚拟DOM；
  • 4 用虚拟DOM的结构生成真实的DOM来显示；
  • 5 state 发生变化
  • 6 数据 + 模版 生成新的虚拟DOM；
  • 7 比较原始虚拟DOM和新的虚拟DOM的区别，找到对应变化的元素进行修改；
  • 8 根据修改后的虚拟DOM,操作真实DOM,改变变化元素的内容；

优点：

  • 1 性能得到极大的提升;
  • 2 可以跨平台根据虚拟DOM进行真实DOM的操作或是根据虚拟DOM进行原生控件的操作(React-Native)；

虚拟DOM diff算法

  • 1 逐层比较DOM
  • 2 发现一层不一样，直接废弃掉旧的节点中的所有内容替换成新的节点内容；

React - 生命周期

挂载阶段：
  • constructor();
  • 构造函数，在创建组件的时候调用一次。
  • componentWillMount() ;
  • 在组件即将被挂载的时候调用一次。
  • render();
  • 渲染组件 。
  • componentDidMount();
  • 再见被挂载完成的时候调用一次，可以在这里使用refs 。
    卸载阶段(组件从页面中移除)：
  • componentWillUnmount();
  • 即将卸载 / 没有卸载完成。
    更新阶段：
  • componentWillReceiveProps(nextProps);
  • 父组件的更新会触发子组件的这个函数。
  • nextProps 父组件更新的时候带来的数据。
  • shouldComponentUpdate(nextProps,nextState);
  • 是否需要更新渲染。
  • Return false / true。
  • componentWillUpdate(nextprops,nextState);
  • 即将更新 。
  • render();
  • 渲染 。
  • componentDidUpdate(prevprops,PrevState);
  • 完成更新 。

    综合小案例：

  • 结合使用 类实现组件化及父子组件通信和子组件生命周期的更新;

  • 组件之间的关联：

  • 1 在父组件上给子组件添加回调函数(操作父组件的状态)。
  • 2 父组件拿到回调后更新自身状态和更新子组件的状态。
  • 3 子组件更新后会产生生命周期，进入生命周期的更新阶段(shouldComponentUpdate(nextProps,nextState))是否需要更新，是否需要重新渲染组件
  • 生命周期传进来的状态和当前的状态不一样的时候，执行更新组件状态。
